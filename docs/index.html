Overview

PolarPvP-Manager is a comprehensive Minecraft plugin that provides players with control over their PvP status while giving administrators powerful tools to enforce PvP in specific zones and through playtime-based mechanics.

Player Control

Simple commands allow players to toggle PvP on or off, giving them control over their combat experience.

Zone Management

Administrators can create forced PvP zones where combat is always enabled, perfect for arenas and special areas.

Playtime Debt

A sophisticated system that enforces periodic PvP based on player activity, preventing indefinite PvP avoidance.

Fully Customizable

Configure messages, timings, and behavior to match your server's unique needs.

Requirements

  • Minecraft Server 1.20 or higher
  • Java 17 or higher
  • Compatible with Bukkit, Spigot, Paper, and Purpur

Installation

  1. Download the latest release JAR file from the GitHub repository
  2. Place the JAR file in your server's plugins/ directory
  3. Start or restart your Minecraft server
  4. The plugin will create a default configuration at plugins/PolarPvP-Manager/config.yml
  5. Customize the configuration as needed
  6. Use /pvpadmin reload to apply changes without restarting

Commands

Player Commands

These commands are available to all players with the pvptoggle.use permission (enabled by default).

Command Description Example
/pvp on Enable your PvP status /pvp on
/pvp off Disable your PvP status /pvp off
/pvp status Check your current PvP status /pvp status

Admin Commands

These commands require the pvptoggle.admin permission (ops by default).

Command Description Example
/pvpadmin wand Get the zone selection wand /pvpadmin wand
/pvpadmin zone create <name> Create a forced PvP zone /pvpadmin zone create arena
/pvpadmin zone delete <name> Remove a forced PvP zone /pvpadmin zone delete arena
/pvpadmin zone list List all zones /pvpadmin zone list
/pvpadmin zone info <name> Display zone details /pvpadmin zone info arena
/pvpadmin player <name> info View player PvP information /pvpadmin player Steve info
/pvpadmin player <name> reset Reset player's PvP data /pvpadmin player Steve reset
/pvpadmin player <name> setdebt <seconds> Set player's forced PvP debt /pvpadmin player Steve setdebt 1200
/pvpadmin simtime <seconds> Simulate playtime for testing /pvpadmin simtime 3600
/pvpadmin reload Reload configuration /pvpadmin reload

Zone Management

Forced PvP zones are defined areas where players cannot disable PvP, regardless of their personal setting. These are perfect for arenas, dungeons, or dedicated PvP areas.

Creating Zones

Step-by-Step Guide

  1. Obtain the zone selection wand using /pvpadmin wand
  2. The wand is a Blaze Rod by default (configurable in config.yml)
  3. Left-click one corner block of your desired zone
  4. Right-click the opposite corner block to complete the selection
  5. Both blocks must be in the same world
  6. Run /pvpadmin zone create <name> to create the zone
Important: Zone selection points must be in the same world. Cross-world zones are not supported.

Managing Zones

  • List zones: /pvpadmin zone list displays all created zones
  • View details: /pvpadmin zone info <name> shows zone boundaries and world
  • Delete zones: /pvpadmin zone delete <name> permanently removes a zone

Playtime Debt System

The playtime debt system automatically enforces periods of mandatory PvP based on player activity. This prevents players from avoiding PvP indefinitely and encourages server engagement.

How It Works

  1. Players accumulate playtime as they play on the server
  2. After a configured number of hours (default: 1 hour), players enter a forced PvP period
  3. During forced PvP, players cannot disable their PvP status
  4. Forced PvP lasts for a configured duration (default: 20 minutes)
  5. Debt only counts down when 2 or more players are online
  6. Logging out does not reduce debt - it persists across sessions

Configuration

Customize the playtime system in your config.yml:

playtime:
  hours-per-cycle: 1        # Hours of playtime to trigger forced PvP
  forced-minutes: 20        # Minutes of forced PvP per cycle
Bypass Permission: Players with the pvptoggle.bypass permission (ops by default) are exempt from playtime debt.

Configuration

The plugin configuration is located at plugins/PolarPvP-Manager/config.yml. All settings can be modified and reloaded without restarting the server.

Key Settings

Setting Description Default Value
default-pvp-state Default PvP state for new players false
playtime.hours-per-cycle Hours between forced PvP periods 1
playtime.forced-minutes Duration of forced PvP in minutes 20
zone-wand-material Material for zone selection wand BLAZE_ROD
save-interval Auto-save interval in minutes 5
debug Enable debug logging false

Message Customization

All messages can be customized in the config file. Use standard Minecraft color codes with the & symbol:

  • &a - Green
  • &c - Red
  • &e - Yellow
  • &7 - Gray
  • &l - Bold
  • &o - Italic

Data Storage

The plugin stores data in YAML files within the plugin directory:

  • playerdata.yml - Player PvP states, playtime, and debt information
  • zones.yml - Zone definitions and boundaries

Data is automatically saved at regular intervals and when the server shuts down.

Permissions

Permission Description Default
pvptoggle.use Allows using /pvp commands All players
pvptoggle.admin Allows using /pvpadmin commands Operators
pvptoggle.bypass Bypass forced PvP from playtime debt Operators

Troubleshooting

Common Issues

PvP toggle not working

  • Verify the player has the pvptoggle.use permission
  • Check if the player is in a forced PvP zone
  • Check if the player has active playtime debt
  • Review configuration settings

Plugin not loading

  • Ensure you are running Java 17 or higher
  • Check for error messages in the server console
  • Verify the plugin JAR is in the correct plugins directory

Zones not saving

  • Verify file permissions on the plugin data directory
  • Check server logs for permission errors
  • Ensure both zone corners are in the same world

Debug Mode

Enable debug logging to troubleshoot issues:

  1. Open config.yml
  2. Set debug: true
  3. Save the file
  4. Reload the plugin with /pvpadmin reload
  5. Check console logs for detailed information

Building from Source

If you want to build the plugin from source code:

Requirements

  • Maven 3.6 or higher
  • Java 17 or higher
  • Git

Build Instructions

# Clone the repository
git clone https://github.com/SSoggyTacoBlud/PolarPvP-Manager.git
cd PolarPvP-Manager

# Build with Maven
mvn clean package

# The compiled JAR will be in target/PolarPvP-Manager-1.0.0.jar
